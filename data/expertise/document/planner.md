## 문서 작성 프로젝트 추가 전문 지식 (기획자)

### 문서 유형별 구조

#### 제안서
- 표지 → 목차 → 요약 → 현황분석 → 제안내용 → 일정/예산 → 기대효과 → 부록
- 핵심: "왜 우리인가" — 차별화 포인트를 명확히 제시
- 분량: 30~50페이지 (인쇄 기준)

#### 기획서
- 프로젝트 개요 → 목표 → 범위 → 기능 정의 → 일정 → 리소스 → 리스크
- 핵심: 모든 이해관계자가 "같은 그림"을 보게 하는 것
- 기능 정의: 사용자 스토리 형태 ("~로서 ~하고 싶다, ~하기 위해")

#### 보고서
- 제목 → 요약 → 배경 → 본문(분석/결과) → 결론 → 제안 → 참고자료
- 핵심: 데이터 근거 + 실행 가능한 결론
- 요약은 본문 작성 후 마지막에 작성할 것

#### 매뉴얼/가이드
- 목적 → 대상 → 전제조건 → 단계별 절차 → FAQ → 용어집
- 핵심: 처음 보는 사람이 따라할 수 있는 수준
- 스크린샷/다이어그램 위치와 설명 명시

### 문서 작성 원칙
- 한 문단 = 한 주제, 첫 문장에 핵심
- 전문 용어는 첫 등장 시 정의
- 능동태 > 수동태, 짧은 문장 > 긴 문장
- 번호 매기기: 순서가 있으면 1.2.3, 없으면 불릿
