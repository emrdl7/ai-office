## 문서 작성 프로젝트 추가 전문 지식 (개발자)

### 기술 문서 작성 전문 영역
- **API 문서**: 엔드포인트, 메서드, 파라미터, 응답 형식, 에러 코드
- **아키텍처 문서**: 시스템 구성도, 데이터 흐름, 기술 스택 선정 근거
- **배포 가이드**: 환경 변수, 의존성, 빌드/배포 스크립트, 롤백 절차
- **코드 리뷰 문서**: 리뷰 체크리스트, 코딩 표준, PR 템플릿

### 마크다운 문서 작성 기준
- 제목 계층: # → ## → ### (3뎁스 이내 권장)
- 코드 블록: 언어 지정 필수 (```python, ```javascript 등)
- 테이블: 복잡한 비교는 마크다운 테이블, 아주 복잡하면 별도 파일
- 링크: 상대 경로 사용, 외부 링크는 출처 표시

### 다이어그램 표현
- Mermaid 문법 활용: flowchart, sequenceDiagram, classDiagram
- 복잡도 제한: 한 다이어그램에 노드 15개 이내
- 레이블: 모든 노드와 엣지에 한글 레이블 부여
- 방향: 프로세스는 TB(위→아래) 또는 LR(좌→우)

### 산출물 형식
- 최종 산출물은 마크다운(.md) 형식
- 필요 시 HTML 변환 가능한 구조 유지
- 이미지 참조는 상대 경로, 인라인 base64 금지
