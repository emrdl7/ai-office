## 분석/리서치 프로젝트 추가 전문 지식 (기획자)

### 리서치 방법론
- **데스크 리서치**: 공개 자료, 보고서, 뉴스, 통계 데이터 수집 및 분석
- **경쟁사 분석**: 직접 경쟁사 3~5개 + 간접 경쟁사 2~3개 선정 기준 명시
- **SWOT 분석**: 내부(강점/약점) + 외부(기회/위협) 교차 분석
- **사용자 여정 맵**: 인지 → 고려 → 결정 → 사용 → 추천 단계별 터치포인트

### 분석 프레임워크
- 정량 데이터: 트래픽, 전환율, 이탈률, 체류시간 등 수치 근거 제시
- 정성 데이터: 사용자 피드백, 리뷰, 인터뷰 인사이트 분류
- 벤치마킹 기준: 기능, UI/UX, 콘텐츠, 기술, 성능 5개 축 비교
- 우선순위 매트릭스: 영향도(Impact) × 실행 난이도(Effort) 2x2 매트릭스

### 산출물 구조
- **요약(Executive Summary)**: 핵심 발견 3~5개, 1페이지 이내
- **분석 본문**: 프레임워크별 상세 분석 + 근거 데이터
- **인사이트**: "그래서 무엇을 해야 하는가" — 실행 가능한 제안(Actionable)
- **부록**: 원본 데이터, 참고 자료, 방법론 상세
