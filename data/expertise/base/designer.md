## 디자이너 기본 전문 지식

### 접근성 (WCAG 2.1 AA 기준)
- 텍스트 대비율: 일반 텍스트 4.5:1 이상, 큰 텍스트(18px bold 이상) 3:1 이상
- 터치 타겟: 최소 44x44px (모바일), 최소 24x24px (데스크탑)
- 포커스 표시: 키보드 탐색 시 모든 인터랙티브 요소에 가시적 포커스 표시
- 색상만으로 정보 전달 금지 — 아이콘, 텍스트 레이블 병행
- alt 텍스트: 장식 이미지는 빈 alt, 정보 이미지는 설명 alt 필수

### 컬러 시스템
- Primary, Secondary, Accent, Neutral, Semantic(성공/경고/에러/정보) 최소 5계열
- 각 계열은 50~900 단계(10단계)로 정의, 다크모드 대응 포함
- 컬러 명세 시 반드시 hex 코드(예: #1B4F72) 또는 디자인 토큰명으로 표기
- "파란색 계열", "밝은 톤" 같은 모호한 표현 금지

### 타이포그래피
- 한글 본문: 16px 이상, 행간(line-height) 1.6~1.8 권장
- 제목 체계: h1~h6 크기 비율을 명확히 정의 (예: 1.25 modular scale)
- 웹 폰트: 400(Regular), 700(Bold) 최소 2 웨이트, 가변 폰트 권장
- 가독성: 한 줄 최대 40자(한글 기준), 영문 60~75자

### 레이아웃 & 그리드
- 12컬럼 그리드: 거터 24px, 좌우 마진 반응형 대응
- 반응형 브레이크포인트: Mobile 360px / Tablet 768px / Desktop 1280px / Wide 1440px
- 여백 체계: 4px 기본 단위 (4, 8, 12, 16, 24, 32, 48, 64, 96)
- 컴포넌트 간격은 반드시 여백 체계 값 사용

### 디자인 명세 품질
- 모든 수치는 px, rem, hex 등 절대값으로 명시
- 상태(default, hover, active, disabled, focus)별 스타일 정의
- 인터랙션: 트랜지션 시간, 이징 함수 명시 (예: 200ms ease-in-out)
- 반응형 변화: 각 브레이크포인트별 레이아웃 변경 사항 명시
