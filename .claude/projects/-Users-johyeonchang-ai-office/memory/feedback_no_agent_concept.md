UI, 카피, 위젯, 브랜딩 어디서든 "에이전트", "팀원", "사무실", "5명 팀" 같은 개념을 드러내지 말 것.

**Why:** 이 프로젝트는 커밋 `4ba1af0`에서 UX Studio(Job 파이프라인 중심)로 피벗했다. "사무실 에이전트 자율 루프" 개념은 제거됐고 사용자는 이 사실을 **여러 세션 반복해 강조**했다. 그럼에도 내가 UI 리디자인에서 "5-agent studio" 부제, TeamStatusWidget, `/api/agents` 사이드바 위젯 같은 에이전트 중심 요소를 다시 넣어 사용자가 심하게 답답해했다.

**How to apply:**
- 사이드바·헤더·빈 상태·브랜딩 카피에서 "agent/에이전트/팀원/팀장·기획·디자인·개발·QA 묶음 표시" 금지
- `/api/agents` 를 사이드바·대시보드 상시 위젯에 붙이지 말 것 (채팅 메시지 발화자 표기는 OK — 메시지 본문의 화자일 뿐)
- 대신 Job/Playbook/Step/산출물/게이트/비용 중심 UI를 쓸 것
- 애매하면 project_architecture.md 메모리 확인 후 진행
- 이 문서의 존재 자체가 "또 반복하지 말라"는 경고 — 새 UI 요소 넣기 전에 이 경고를 다시 읽을 것
