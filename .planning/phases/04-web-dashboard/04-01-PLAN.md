  <name>Task 1: 백엔드 API 확장 (DAG, 파일, 에이전트, 로그 히스토리)</name>
  <files>server/main.py</files>
  <action>
    server/main.py에 다음 엔드포인트 추가:

    1. GET /api/dag
       - OrchestrationLoop의 TaskGraph.to_state_dict()를 읽어 React Flow 형식으로 변환
       - 노드: {id, data: {label, status, assigned_to, artifact_paths}, position: {x, y}}
       - 엣지: {id, source, target}
       - 위치는 간단한 레이아웃 계산 (depends_on depth 기반)

    2. GET /api/files/{task_id}
       - workspace/{task_id}/ 내 파일 목록 반환
       - [{path: relative_path, type: code|doc|design|data|unknown, size: bytes}]
       - WorkspaceManager(task_id=task_id)로 list_artifacts() 호출
       - task_id가 '..'나 '/' 포함 시 400 반환

    3. GET /api/files/{task_id}/{file_path:path}
       - 파일 내용을 text/plain으로 반환
       - WorkspaceManager.safe_path()로 경로 순회 방지
       - 파일 없으면 404

    4. GET /api/agents
       - 에이전트별 상태 목록 반환
       - active_tasks에서 추론: [{agent_id, status, current_task_id}]
       - 알려진 에이전트: planner, designer, developer, qa, claude

    5. GET /api/tasks
       - active_tasks 딕셔너리 전체를 리스트로 반환
       - [{task_id, state, created_at}] 형식 (created_at은 app 시작 후 등록 순서)
       - task_order 리스트를 app.state에 추가 (POST /api/tasks 시 순서 기록)

    6. GET /api/logs/history
       - 최근 N건(기본 100) 로그를 반환 (새로고침 후 WebSocket 전 복구용)
       - event_bus에 log_history: list[LogEvent] 추가 (최대 500건 순환 버퍼)
       - EventBus에 history 관리 기능이 없으면 main.py의 app.state.log_history로 관리

    CORS 설정:
    - CORSMiddleware 추가 (localhost:5173 허용) — Vite dev 서버 연동

    app.state 확장:
    - task_order: list[str] (작업 지시 순서 추적)
    - log_history: list[dict] (최근 로그 버퍼, 최대 500건)

    POST /api/tasks 수정:
    - task_order에 task_id 추가

    WebSocket /ws/logs 수정:
    - 이벤트를 log_history에도 추가 (최대 500건 유지)
  </action>
