    1. CORSMiddleware (대시보드 Vite dev 서버 연동):
       - from fastapi.middleware.cors import CORSMiddleware
       - app.add_middleware(CORSMiddleware, allow_origins=["http://localhost:5173"], allow_methods=["*"], allow_headers=["*"])

    2. app.state 확장 (lifespan 함수 내):
       - task_order: list[str] = []  # 작업 지시 순서 추적
       - log_history: list[dict] = []  # 최근 로그 버퍼 (최대 500건)

    3. POST /api/tasks 수정:
       - task_order에 task_id 추가

    4. WebSocket /ws/logs 수정:
       - 이벤트를 log_history에도 추가 (최대 500건 순환)

    5. GET /api/tasks (신규):
       - active_tasks 딕셔너리를 task_order 순서대로 리스트로 반환
       - 응답: [{"task_id": str, "state": str}]

    6. GET /api/agents (신규):
       - 알려진 에이전트 목록과 상태 반환
       - KNOWN_AGENTS = ["claude", "planner", "designer", "developer", "qa"]
       - 현재는 정적으로 idle 상태 반환 (WebSocket 이벤트로 실시간 업데이트는 프론트엔드 책임)
       - 응답: [{"agent_id": str, "status": str}]

    7. GET /api/logs/history (신규):
       - app.state.log_history 반환 (최근 N건, 기본 100)
       - query param: limit (기본 100, 최대 500)
       - 응답: list[dict]

    8. GET /api/workspace/{task_id}/files (신규):
       - task_id 검증: ".." 또는 "/" 포함 시 400
       - WorkspaceManager(task_id=task_id)로 list_artifacts() 호출
       - workspace/{task_id}/ 디렉토리가 없으면 빈 목록 반환
       - 응답: [{"path": str, "type": str, "size": int}]

    9. GET /api/workspace/{task_id}/files/{file_path:path} (신규):
       - task_id 검증: ".." 또는 "/" 포함 시 400
       - WorkspaceManager.safe_path()로 경로 순회 방지
       - 파일 없으면 404
       - 응답: text/plain으로 파일 내용 반환

    10. GET /api/dag (신규):
        - OrchestrationLoop의 task_graph.to_state_dict() 반환
        - 빈 그래프면 {"nodes": [], "edges": []} 반환
        - to_state_dict() 결과를 React Flow 형식으로 변환:
          nodes: [{id, data: {label, status, assigned_to, artifact_paths}, position: {x, y}}]
          edges: [{id, source, target}]
        - 위치는 topological sort 기반 레이아웃 (x: depth*250, y: index*100)
