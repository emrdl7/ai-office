    **Step 1: server/orchestration/loop.py 생성**

    **WorkflowState enum (str, Enum):**
    ```
    IDLE, CLAUDE_ANALYZING, PLANNER_PLANNING, WORKER_EXECUTING,
    QA_REVIEWING, CLAUDE_FINAL_VERIFYING, REVISION_LOOPING,
    COMPLETED, ESCALATED
    ```

    **OrchestrationLoop 클래스:**
    ```python
    class OrchestrationLoop:
        MAX_REVISION_ROUNDS = 3  # D-11

        def __init__(
            self,
            bus: MessageBus,
            runner: OllamaRunner,
            event_bus: EventBus,
            workspace: WorkspaceManager,
            router: MessageRouter,
        ):
            self.bus = bus
            self.runner = runner
            self.event_bus = event_bus
            self.workspace = workspace
            self.router = router
            self._state = WorkflowState.IDLE
            self._revision_count = 0
            self._task_graph: TaskGraph | None = None
    ```

    **메서드 구현:**

    `async def run(self, user_instruction: str) -> WorkflowState`:
    전체 오케스트레이션 루프 실행. 상태 전이 순서:
    1. IDLE → CLAUDE_ANALYZING: analyze_instruction(user_instruction)
    2. CLAUDE_ANALYZING → PLANNER_PLANNING: planner 메시지 소비 → _run_planner()
    3. PLANNER_PLANNING → WORKER_EXECUTING: ready_tasks() 순회 → 각 작업자 실행
    4. WORKER_EXECUTING → QA_REVIEWING: 각 작업자 완료 후 _run_qa_gate(node)
    5. QA_REVIEWING → CLAUDE_FINAL_VERIFYING (all_done()) or 재작업 루프
    6. CLAUDE_FINAL_VERIFYING → COMPLETED | REVISION_LOOPING
    7. REVISION_LOOPING: revision_count 체크 → MAX 초과 시 ESCALATED, 아니면 PLANNER_PLANNING 재진입

    각 상태 전이마다 event_bus.publish(LogEvent(agent_id='orchestrator', event_type='status_change', message=f'상태: {self._state}')) 발행.

    `async def analyze_instruction(self, instruction: str) -> AgentMessage`:
    - 프롬프트: "다음 사용자 지시를 분석하여 기획자에게 전달할 작업 방향을 JSON으로 응답하라: {instruction}"
    - run_claude_isolated(prompt) 호출
    - 응답을 파싱하여 AgentMessage(type='task_request', from_agent='claude', to_agent='planner', payload=...) 생성
    - router.route(msg) 발행 후 msg 반환

    `async def _run_agent(self, node: TaskNode) -> TaskResultPayload | None`:
    - agents/{node.assigned_to}.md 파일 읽어 system prompt 로드
    - 프롬프트: "[작업 지시]\n{node.description}\n\n[원본 요구사항]\n{node.requirements}"
    - runner.generate_json(prompt, system=system_prompt) 호출
    - 응답을 TaskResultPayload로 파싱 (없으면 fail)
    - workspace.write_artifact(task_id=node.task_id, content=result, filename='result.json') 저장
    - task_graph.update_status(node.task_id, TaskStatus.DONE, artifact_paths=result.artifact_paths) 업데이트

    `async def _run_qa_gate(self, node: TaskNode) -> bool`:
    - agents/qa.md 파일 읽어 system prompt 로드
    - 프롬프트 (Pattern 4 — 확증편향 방지):
      ```
      [원본 요구사항]
      {node.requirements}

      [작업 결과물 경로]
      {node.artifact_paths}

      위 원본 요구사항을 기준으로 작업 결과물을 검수하라.
      결과물만 보고 판단하지 말고 반드시 원본 요구사항 대비 검증하라.
      ```
    - runner.generate_json(prompt, system=system_prompt) 호출
    - 결과의 status == 'success' 이면 True, 아니면 node.failure_reason = result.failure_reason, False

    `async def _claude_final_verify(self, task_graph: TaskGraph) -> bool`:
    - 모든 완료 태스크의 artifact_paths를 수집하여 Claude에게 검증 요청
    - 프롬프트: "[산출물 목록]\n{artifact_summary}\n\n위 산출물이 원본 요구사항을 충족하는지 검증하라. PASS 또는 FAIL과 함께 이유를 명시하라."
    - run_claude_isolated(prompt) 호출
    - 응답에 'PASS' 포함 시 True, 'FAIL' 포함 시 False (대소문자 무관 확인)
